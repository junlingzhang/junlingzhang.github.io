# 简单配置

# 全局安装

npm install -g vuepress

# 新建文件夹

mkdir vuepress-test

# 初始化

npm init -y

package.json增加脚本

{
  "scripts": {
    "docs:dev": "vuepress dev docs",
    "docs:build": "vuepress build docs"
  }
}

# 新建README.md 可作为首页

---
home: true
heroImage: /favicon.ico
actionText: 快速上手 →
actionLink: /guide/
features:
- title: 简洁至上
  details: 以 Markdown 为中心的项目结构，以最少的配置帮助你专注于写作。
- title: Vue驱动
  details: 享受 Vue + webpack 的开发体验，在 Markdown 中使用 Vue 组件，同时可以使用 Vue 来开发自定义主题。
- title: 高性能
  details: VuePress 为每个页面预渲染生成静态的 HTML，同时在页面被加载的时候，将作为 SPA 运行。
footer: MIT Licensed | Copyright © 2018-present xxxxxx
---

# 新建 docs

# docs中新建 .vuepress文件夹

# .vuepress文件夹新建config.js

module.exports = {
  title: '我是标题',
  description: '我是描述',
  head: [//其中路径的"/"就是public资源目录；
      ['link', { rel: 'icon', href: './1.jpg' }]
  ],
  dest: './docs/.vuepress/dist',
  ga: '',
  evergreen: true,
  themeConfig: {
    nav: [
      { text: '首页', link: '/' },
      { text: '菜单1', link: '/a' },
      {
        text: '菜单2',
        items: [
          { text: '2-1', link: '/b/1' },
          { text: '2-2', link: '/b/2' }
        ]
      },
      { text: '百度', link: 'https://www.baidu.com' },
    ],
    sidebarDepth: 2,
    sidebar:'auto'//auto代表自动生成，也可自定义配置；
  }
}

# .vuepress文件夹新建public文件夹

此文件夹主要放静态资源文件，例如favicons和 PWA的图标。

# 启动

yarn docs:dev

# 构建

yarn docs:build

build生成静态的HTML文件,默认会在 .vuepress/dist 文件夹下

###总体目录结构：

vuepress-test
├─── docs
│   ├── README.md
│   └── .vuepress
│       ├── public
│       └── config.js
└── package.json

# 一键发布到Github

# github新建仓库

# 新建仓库一： USERNAME.github.io

负责显示网站内容，我们不需要改动它 要保证和Github账号名一模一样！

# 新建仓库二： blogName

日常开发和新增内容都在该仓库

# 根目录下新建.sh文件

#!/usr/bin/env sh

# 确保脚本抛出遇到的错误
set -e

# 生成静态文件
npm run docs:build

# 进入生成的文件夹
cd docs/.vuepress/dist

# 如果是发布到自定义域名
# echo 'www.example.com' > CNAME

git init
git add -A
git commit -m 'deploy'

# 如果发布到 https://<USERNAME>.github.io
# git push -f git@github.com:<USERNAME>/<USERNAME>.github.io.git master

# 如果发布到 https://<USERNAME>.github.io/<REPO>
# git push -f git@github.com:<USERNAME>/<REPO>.git master:gh-pages

cd -

# 设置package.json

{
    "scripts": {
        "publish": "bash publish.sh"
      },
}

TIP

运行npm run deploy 即可自动构建部署到github上。
